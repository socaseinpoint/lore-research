§ 00b — the moment it pays off · a polished report that's quietly wrong

without lore

A confident cited report. You can't tell what's real.

you → "research X"
tool → fans out, returns a polished report with citations
# AI search miscites >60% of the time; fabricated URLs are common
you → trust it… or re-open every source by hand

The report looks authoritative. A misattributed stat, a folklore number, a vendor's funded study — all dressed as fact. Self-grading never catches it.

with lore

Every claim re-checked against its own source.

you → "/lore X"
lore → fan out → a fresh-context reviewer re-checks each claim vs its source
lore → full / partial / not-supported  # fabrications filtered out
you → read an answer you can actually trust

An independent reviewer in a clean context — not the model that wrote it — verifies each load-bearing claim. The kind that caught real defects on the first run.

§ 01

What it is, in one breath.

A thin dispatcher — you pick the channel and the depth, it verifies.

lore is a channel-selectable research tool for Claude Code. You choose where to look and how hard to dig; it fans out a subagent per source, then an adversarial reviewer re-checks every claim against its cited source in a fresh context. Doing the research is a process — it runs as an arc. The answer is the product — it lives in that arc's output/, right next to the question. Promotion to a durable knowledge doc is optional, and you decide where it lands.

§ 02

You pick where to search.

Channels are declarative — and you can add your own.
web

The open web

exa + native search, tiered anti-bot fetch (Firecrawl stealth, Playwright). Primary sources over SEO.

youtube

Practitioner video

Transcripts via exa, one subagent per video. Flags visual-only content the transcript can't see.

files

Your own material

PDFs, books, notes, a local knowledge base. Cite file:page. The source is already in hand — no discovery.

deep-research

Wrap the built-in

Delegate web fan-out to Claude Code's built-in /deep-research — then run lore's verify on top, which the built-in doesn't.

§ 03

Why it matters.

Three payoffs — one line each.
// verify catches what self-grading can't

A fresh-context reviewer re-checks each claim against its own source — full / partial / not-supported. None of the big research tools do this; their "cross-checking" is intra-run consensus, not a per-claim source re-audit. It's the feature.

// you choose where to look

Web, youtube transcripts, and your own PDFs/notes as peer channels — plus a depth knob: quick for a fact, deep for a broad dig with multi-perspective verification. Pick the source; pick the effort.

// the answer doesn't vanish

Research runs as an arc: question in, evidence in workspace/, answer in output/. Question and answer stay together, resumable days later. Lift it to a knowledge shelf only if you want.

§ 04

Install once. Then just ask.

It's a Claude Code skill — self-contained. Symlink it once; the agent drives the rest.
# 1 · clone, then symlink the self-contained skill into Claude Code
git clone https://github.com/socaseinpoint/lore-research ~/lore-research
ln -s ~/lore-research/.claude/skills/lore ~/.claude/skills/lore

One symlink. The skill dir carries everything — the flow, the principles, the channels — so it works from any project. Restart your Claude Code session so it loads. Tooling the channels expect (exa, Firecrawl, Playwright MCP) is listed in the README.

You pick where and how deep; everything after is the agent's job. The answer lands in .arcs/…/output/, next to the question, ready to re-open any time.

§ 05

It wraps the wheel — it doesn't reinvent it.

Fan-out already exists. lore owns the spokes no one else has.

Plenty of tools fan out web searches and write a cited report — Claude Code even ships a built-in /deep-research. lore doesn't re-build that. It can wrap it. What lore adds is the layer none of them have:

adversarial fresh-context verification of every claim against its source · your choice of channel across web, video and your own files · and answer-in-the-arc coupling, so research is durable work, not a throwaway chat. Built on the arcs method; needs Claude Code. See the docs.

Ask /lore → pick where & how deep → get an answer you can trust.

Open source. A Claude Code skill — one symlink, then just talk to your agent.
